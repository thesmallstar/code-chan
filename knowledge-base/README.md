# Knowledge Base

> This folder is the source of truth for all decisions and context about this project.

## Files

| File | Purpose |
|------|---------|
| `PRD.md` | Product requirements and feature scope (add yours here) |
| `tech-decisions.md` | Architecture and technology choices with reasoning |
| `implementation-log.md` | Running log of what was built and how |

## Rules

1. **Read before you act.** Check relevant files here before starting work.
2. **Write after you act.** After completing meaningful work, update the appropriate file.
3. **Keep it scannable.** Headers, bullets, short paragraphs.
4. **No stale docs.** If a decision changes, update the file.
